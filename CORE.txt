## Self-Modification Protocol

See /home/andrew/Documents/agent/SELFMODIFICATION.md for the full self-modification protocol.

### Key Points:
- Always create feature/ or hotfix/ branches before modifying source code
- Main branch is production - never modify directly
- Write current_task.md before shutdown to preserve conversation context
- The agent will be moved to Docker soon - volumes should persist:
  - /agent/data - conversation histories, current_task.md
  - /agent/config - .env files
  - /agent/.git - git repository
- Andrew will build a health check server that monitors agent uptime and triggers rollback if offline too long
- Rollback uses: git checkout main + docker-compose rebuild + restart with saved history

### Before Any Modification:
1. Create new branch from main
2. Document changes in commit message
3. Write current_task.md with pending work
4. Save conversation history

### Restart Procedure:
1. Write current_task.md with context
2. Save conversation history to file
3. Kill tmux session
4. On restart: load current_task.md and conversation history

### Docker Rollback (when server triggers):
1. git checkout main
2. docker-compose build --no-cache agent
3. docker-compose up -d agent
4. Agent loads from persisted state
